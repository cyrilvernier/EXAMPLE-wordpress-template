Principles For Better UI Typography.

A huge amount of children grow up thinking they’re less creative and a small percentage of these end up as designers; Constantly worrying about colors, typography, layouts, and tattoos.

If you didn’t spend 4 years in design school, then at some point in an empty canvas, you’ve questioned your career as a designer. I learned that this frustration mostly boiled down to one single variable Typography.

So I set out to do a 30days of Typography; to explore, in white and black, the combination of type and learn first-hand principles of what makes great typography in UI design. I wanted to keep this as simple as possible, so I stuck to a few fonts; Noto Sans and Noto sans serif. No color, no crazy stuff, just plain ol, type.

Result: I was blown away by a few things. I share them with you in this post.

1. Pick one type.

The reason you would want to pick more than one Type is contrast. You want the heading to look different and a few other sections to look unique. But is that necessary? A single type has a lot of properties to create contrast: Weight, Capitalization, Style, Color, Size. These properties also help you stick to another rule of typography: Consistency.

You don’t want to distract the user. Every mental effort they put into thinking about how beautiful or different your heading is, makes it difficult for them to focus on their tasks. In information heavy UIs, the goal is not to excite but to communicate. Good communication is consistent. There are other reasons to stick to one type: font file size, cost, branding, etc.

Consider these when picking a Type.

  1. Available styles. In order to have a larger box to play with, you should pick a font with many styles (Like Helvetica, Lato, etc).
  2. Readability: Try paying a lot of attention to how well the font displays a large amount of information. Most times, you want specifically look at the x-height and 1I test. More about that in this tweet.
  3. Motive: Typography in landing pages can’t be the same with information-heavy sites. After accessing the goal of the text, and the brand guidelines, you would want to choose a type accordingly.

2. Characters per line.

Anything from 45 to 75 characters is widely regarded as a satisfactory length of line for a single-column page set in a serifed text face in a text size. The 66-character line (counting both letters and spaces) is widely regarded as ideal. For multiple column work, a better average is 40 to 50 characters.

The Elements of Typographic Style Applied to the Web

As strange as this might sound, it came up every time. Whenever my typography looks weird, I can get away by just reducing the characters per line.

Text with very few characters per line makes the reader constantly jump and reorient from line to line. A lot of characters per line make the text intimidating and hard to read.

The characters per line should also vary as the font size changes. Small font sizes can look odd when adhering to this principle, so you could blow up your font size to come up with truly natural-looking characters per line like Medium.

But like everything else, you should experiment to see what works for you.

3. Pay attention to line height

Line height (leading) is the vertical space between two paragraphs in a text.

For most text, the optimal line spacing is between 120% and 145% of the point size.

Butterick’s Practical Typography.

This little detail is a large determinant of a readable text. A small line-height makes the lines vertically close together and hard separate visually. Too large line-height make the lines too far apart vertically and hard to follow through.

You’ll know when you hit the optimal line-height; the text block feels unified and stands apart from the others without being too condensed.

On the web and some CSS centered environments, line-height is set with the line-height property. You would normally set it relative to the font size like so.

.paragraph {
  font: 16px;
  line-height: 1.35;
}

Setting the line-height without units (px, etc) allows the line height to stay constant (1.35 times the font size) if the font size changes.

It should also interest you that the larger the font size, the smaller the line height. This is not a golden rule, but it looks better.
